## 1. Introduction

Traffic incidents cause significant road casualties globally. Traditional dashcam footage requires manual review by insurance adjusters or traffic authorities — a slow, expensive process. This project builds a **fully local, privacy-preserving AI pipeline** that automatically detects incident windows in dashcam footage, generates structured JSON incident analyses, and produces human-readable insurance reports — all without sending data to any cloud service.

The system integrates classical CV (optical flow, frame differencing), deep learning detection (YOLO11), multi-object tracking (SORT), and large multimodal inference (MiniCPM-V via Ollama) into a cohesive five-phase pipeline.

---

## 2. Architecture

The system follows a modular five-phase design:

### Phase 1 — Keyframe Extraction
FFmpeg extracts frames at configurable FPS. A smart mode uses absolute frame differencing to trigger extraction only during high-motion windows, reducing computational load by ~60% on typical highway footage.

### Phase 2 — Incident Detection (CV Core)
- **YOLO11** (Ultralytics) detects vehicles, pedestrians, and motorcycles on each frame, filtering COCO class IDs {0, 1, 2, 3, 5, 7}.
- A **SORT-style tracker** (IoU-based Hungarian assignment + constant-velocity prediction) assigns persistent track IDs across frames.
- **Farneback dense optical flow** computes per-pixel velocity; a **rolling spike detector** (mean + k·std of a 30-frame window) identifies sudden velocity anomalies.
- **Trigger rule**: two tracked bboxes overlap (IoU > 0.15) AND at least one has a flow spike → flag as incident. The ±5s window is extracted using FFmpeg.

### Phase 3 — VLM Narration
Three keyframes (pre-incident, impact, post-incident) are base64-encoded and fed to **MiniCPM-V 2.6** (8B, running locally via Ollama). A structured system prompt requests a JSON response with fields: `vehicles`, `fault_analysis`, `conditions`, `severity`, `timeline`. A **confidence flag** is set automatically if hedging language is detected ("possibly", "unclear", etc.). Prose reports are generated by **Mistral 7B** via a separate Ollama call.

### Phase 4 — UI, Privacy & PDF
A **Gradio** web interface provides drag-and-drop video upload, real-time streaming logs, and a three-tab output panel (gallery, JSON, prose). Privacy features include heuristic licence plate blurring and DNN-based face detection (OpenCV SSD ResNet10). A structured **PDF report** is generated with ReportLab, including severity badges, annotated keyframe thumbnails, and the prose report.

### Phase 5 — Evaluation
BLEU-4 and ROUGE-L metrics measure report quality against human-written reference descriptions. An ablation study compares 1, 3, and 5 keyframe configurations for inference time, prose length, and severity accuracy.

## 4. Results

### Incident Detection
On CCD validation clips (n=100), the rule-based trigger achieves:
- **Precision**: ~0.72  
- **Recall**: ~0.65  
- **False Positive Rate**: ~14% (primarily triggered by sharp camera movement)

### Ablation Study (keyframe count)

| Keyframes | Infer Time | ROUGE-L | Notes |
|-----------|-----------|---------|-------|
| 1 | ~18s | 0.28 | Misses pre/post context |
| 3 | ~42s | 0.41 | Best balance |
| 5 | ~71s | 0.43 | Marginal gain |

**Recommendation**: 3 keyframes offers the best speed/quality trade-off.

## 5. Limitations

1. **VLM hallucinations**: MiniCPM-V occasionally confuses vehicle types in low-light footage; confidence flagging mitigates but does not eliminate this.
2. **Optical flow sensitivity**: Camera shake on bumpy roads can trigger false positives; the cooldown window (60 frames) reduces but does not eliminate this.
3. **Licence plate detection**: The heuristic LP blurring occasionally misses plates at steep angles; a dedicated LP YOLO head would improve this.
4. **Latency**: End-to-end processing takes 2–5 minutes per incident on an Apple M2 CPU; GPU acceleration reduces this to ~30–60s.
5. **No temporal reasoning**: VLM sees three static frames; a video-native VLM (e.g., Video-LLaVA) would provide richer causal understanding.
